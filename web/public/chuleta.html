UF02 — OP ART (HTML/CSS/JS)

Que es

Web educativa sobre Op Art (ilusiones opticas geometricas). 7 paginas estaticas. Se abre directo en navegador.

Documentacion UX (por que HTML y no Figma)

"Lo hice en HTML porque asi exporto directo a PDF y practico maquetacion. El resultado es igual que Figma."
  • User Personas = perfiles inventados de usuarios (nombre, edad, que buscan)
  • Journey Maps = recorrido del usuario paso a paso en la web
  • Wireframes = esquemas grises de cada pagina antes de disenarla

Estructura web

  • index.html = home (carousel, secciones, artistas)
  • contingut/principis.html = 3 principios del op art
  • contingut/artistes.html = 4 artistas (Riley, Vasarely, Cruz-Diez, Soto)
  • contingut/demos.html = 4 demos interactivas con sliders
  • contacte.html = formulario de contacto
  • privacitat.html = politica de privacidad
  • assets/css/style.css = todo el estilo visual (2655 lineas)
  • assets/js/main.js = logica general
  • assets/js/demos.js = logica de los 4 patrones

Como funciona cada cosa

  • Carousel hero — JS tiene timer que cada 3s cambia imagen. Puntos de abajo saltan a una concreta.
  • Demos (sliders) — cada slider cambia una variable CSS en tiempo real. Mueves > JS cambia valor > patron se redibuja.
  • Formulario — JS comprueba campos en tiempo real. Sin @ = error rojo. Todo ok = verde.
  • Popup newsletter — JS espera unos segundos y lo muestra. localStorage recuerda si ya lo cerraste.
  • Tarjetas flip — CSS puro. Al pasar raton, rotacion 3D de 180 grados. Sin JS.
  • Menu hamburguesa — 3 rayas en movil. JS cambia clase CSS que muestra/oculta menu.
  • Animaciones scroll — IntersectionObserver detecta elemento en pantalla y activa animacion CSS.

Conceptos tecnicos clave

  • HTML semantico — etiquetas con significado (header, nav, main, footer) en vez de todo div. Ayuda a Google y lectores de pantalla.
  • CSS variables — defines color una vez (--color-primary: #2B4C7E) y lo usas en todo. Cambias uno, cambia todo.
  • Flexbox — alinear cosas en fila o columna. "Pon estos 3 elementos uno al lado del otro."
  • Grid — como Flexbox pero filas Y columnas. Para layouts complejos.
  • Responsive — CSS detecta tamano pantalla. Movil = 1 columna, tablet = 2, PC = 3.
  • Mobile-first — disenar primero para movil, despues anadir reglas para grande.
  • localStorage — memoria del navegador que persiste aunque cierres pagina.
  • aria-labels — atributos para accesibilidad. Dicen al lector de pantalla que es cada cosa.
  • Contraste AA — norma WCAG: texto debe tener contraste suficiente con fondo (4.5:1 minimo).
  • AVIF/WebP — formatos imagen modernos, mas ligeros que JPG/PNG.
  • SVG — imagenes vectoriales (logos, iconos). No se pixelan, pesan poco.

Como se hicieron las memorias (HTML > PDF)

Yo redactaba el contenido y la IA lo maquetaba en HTML; al generar el PDF con Puppeteer, el CSS @media print activa saltos de pagina automaticos para que cada seccion empiece en hoja nueva. Elegi este metodo porque me permite editar y regenerar sin tocar el PDF a mano.
